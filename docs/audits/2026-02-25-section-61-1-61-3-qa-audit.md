## 3. UI/UX best practices (strict)

### 3.1 Touch targets (Apple HIG: minimum 44pt)

| Element | Result | Notes |
|---------|--------|------|
| “Yes, doing it now” / “No, I can’t do it now…” buttons | **Fail** | Inherit `.job-confirm-create-new` / `.job-confirm-add-btn` with `min-height: 40px`. Below 44pt recommendation. |
| Co-seller `<select>` | **Fail** | `.doing-it-now-coseller-select` has `min-height: 40px`. Below 44pt for primary interactive controls. |

**Recommendation:** Increase to at least 44px (e.g. `min-height: 44px`) for these controls on mobile, or scope a mobile-only rule under `body[data-viewport-mode="mobile"]` so desktop is unchanged.

### 3.2 Long copy and wrapping

| Check | Result | Notes |

## 4. Logic and lifecycle gaps

### 4.1 Co-seller list and current user

| Check | Result | Notes |
|-------|--------|------|
| Current user excluded from co-seller dropdown | **Fail** | Exclusion uses `authState.user?.id`. If `authState.user` is null (e.g. session present but user object not set), `selfId` is `''` and the filter `t.user_id !== selfId` does not remove the current user. Dropdown can then include the logged-in technician. |

**Recommendation:** Derive current user id from a fallback (e.g. from token or session) when `authState.user` is null, or treat “no self id” as “exclude no one” and document the edge case.

### 4.2 Quote modal closed while “Doing it now?” is open

| Check | Result | Notes |
|-------|--------|------|
| Promise and modal stack when quote is closed during flow | **Fail** | `#doingItNowModal` and `#jobConfirmOverlay` are inside `#quoteModal`. If the user closes the quote modal (e.g. “‹ Diagram” or close button) while the doing-it-now modal is open, `closeAccessibleModal('quoteModal')` only removes the quote modal from the stack and hides it. The overlays are hidden by DOM containment, but: (1) `jobConfirmOverlay` and `doingItNowModal` may remain in the modal stack, and (2) the Promise from `showDoingItNowModal()` never resolves, so `handleCreateNew` remains stuck at `await showDoingItNowModal(...)`. No cleanup or rejection. |

**Recommendation:** On quote modal close, either close all modals that are logically children (e.g. job confirm, doing-it-now) and reject/resolve any pending doing-it-now Promise, or have the doing-it-now flow observe quote modal close and resolve/reject the Promise so `handleCreateNew` does not hang.

### 4.3 Listener cleanup

| Check | Result | Notes |
|-------|--------|------|
| Yes/No click listeners removed on close | **Pass** | `closeWith` removes both listeners before resolving. |
| Listeners removed if modal is closed by any other path | **Fail** | Listeners are only removed when the user clicks Yes or No. If the modal is ever closed by another code path (e.g. future “close all” or parent close that explicitly closes `doingItNowModal`), the listeners would not be removed and the Promise might never resolve. Tied to finding 4.2. |

## 6. Bug / gap list (no code changes)

1. **Touch targets (61.2/61.3):** “Doing it now?” buttons and co-seller select use 40px min-height; below Apple HIG 44pt. Fails strict UI/UX for mobile.
2. **Long button label (61.2):** “No, I can’t do it now — please get the office to schedule” can wrap badly on narrow viewports; UX and consistency with “Yes” button need review.
3. **Co-seller list (61.3):** When `authState.user` is null, current user is not excluded from the dropdown; they can appear as a co-seller option.
4. **Quote modal closed during flow:** Closing the quote modal while the “Doing it now?” modal is open leaves the Promise from `showDoingItNowModal()` unresolved and can leave the modal stack with stale entries; `handleCreateNew` can hang.
5. **Listener cleanup:** Yes/No listeners are only removed on button click; any other close path (e.g. parent close or future “close all”) does not remove them or resolve the Promise.
