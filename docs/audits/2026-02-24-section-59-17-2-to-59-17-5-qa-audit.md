## 3. Bugs, missing cleanup, and logic gaps

### 3.1 Edit job form: Enter key can submit form (potential page reload)

**Type:** Bug / UX  
**Where:** `#bonusAdminEditJobForm` (index.html), no submit handler in app.js  

**Detail:** The edit job modal uses a `<form id="bonusAdminEditJobForm">` with several inputs. The Save and Cancel buttons are `type="button"`, so they do not submit the form. In many browsers, pressing **Enter** in a text or number input inside a form triggers **form submit**. This form has no `action` and no `submit` listener, so submit may cause a **full page reload**, losing state and closing the modal in a destructive way.

**Expected:** Either prevent form submit (e.g. `form.addEventListener('submit', (e) => { e.preventDefault(); saveBonusAdminEditJob(); })`) or remove the `<form>` wrapper and use a plain `<div>` for layout so Enter does not trigger submit.

**Do not fix yet:** Awaiting approval per audit instructions.

---

### 3.2 Bonus Admin edit job modal not closed when redirecting away from Bonus Admin

**Type:** Logic gap / cleanup  
**Where:** `syncAdminDesktopAccess()` in app.js (e.g. lines 12987–13000)  

**Detail:** When the user is on `view-bonus-admin` and loses desktop admin access (e.g. viewport resized to mobile, or role changed), the code calls `switchView('view-canvas', …)` and shows a message. It does **not** call `closeAccessibleModal('bonusAdminEditJobModal')` or `closeAllModals()`. If the user had the **Edit job** modal open when the resize (or role change) happened, the modal remains open on top of the canvas view. The modal is only relevant in the Bonus Admin context; leaving it open after redirect is confusing and can leave `bonusAdminEditJobId` set.

**Expected:** When redirecting from `view-bonus-admin` to `view-canvas` (or to login) due to `!canAccessDesktopAdminUi()`, close the Bonus Admin edit job modal if it is open (e.g. call `closeAccessibleModal('bonusAdminEditJobModal')` before or after `switchView`, or include it in a targeted cleanup), and clear `bonusAdminEditJobId`.

**Do not fix yet:** Awaiting approval per audit instructions.
